# Simplifier Critique — markthisdown iteration 1

## Pass 1: Catalog and challenge

**16 ACs, ~55 sub-ACs** for a brownfield baseline iteration. That is the first problem.

AC13–AC16 are pure [adopted] regression coverage — 21 sub-ACs describing behavior that already ships. They add no delta value and exist only to pad the testable surface. If the code is already working, these ACs are documentation, not a spec. Cut them or move to a separate "baseline register" document that does not inflate the iteration's AC count.

AC3 (sidebar search removed) and the "Removed" section both list the same item. Removing a feature should not generate two ACs; one AC with two sub-ACs is already one too many. Cut to a single sub-AC: "sidebar search bar is absent."

AC5 (margin hit-test no-op) is a removal of a fallback. The AC tests that something does not exist. One sub-AC is sufficient; the [adopted] tag on one sub-AC suggests the second is redundant regression cover.

## Pass 2: Over-specification

AC12 (Outline view) has 5 sub-ACs but the Open Questions section admits placement is deferred. You cannot write 5 ACs for a feature whose UI placement is undefined. Either defer the feature entirely or reduce to 2 ACs: "headings appear in outline" and "clicking jumps to heading." The remaining 3 are premature.

AC6 (Frontmatter collapse, 4 sub-ACs) has the same problem: "trigger mechanism not specified" is listed as an open question, yet 4 ACs describe collapse behavior. Collapse the ACs to 2 until the trigger is decided.

AC10 (file-path/bracket-tag coloring) and AC11 (code-block highlighting) both carry undefined heuristics in Open Questions. Shipping ACs against undefined detection logic means acceptance is untestable. These should be blocked behind resolving the open questions, not listed as delta ACs.

## Pass 3: Minimum viable cut list

Remove to ship in half the time:
- **AC13–AC16** (adopted regression, 21 sub-ACs) — move to a baseline register, out of the iteration
- **AC12** reduce to 2 sub-ACs; defer placement ACs
- **AC6** reduce to 2 sub-ACs pending trigger decision
- **AC3** reduce to 1 sub-AC
- **AC5** reduce to 1 sub-AC

That removes roughly 25 sub-ACs and unblocks the iteration from open questions it cannot yet answer.

## Pass 4: Migration is "None" — noted

No migration cost. The iteration is still too wide: 4 Added features plus 7 Modified behaviors in one pass. A spec with this many open questions should split Added features (tables, outline, highlighting, coloring) into a follow-on iteration and ship the bug-fix Modified items alone first.
